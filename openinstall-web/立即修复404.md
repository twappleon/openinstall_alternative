# 立即修复 404 问题

## 🎯 问题根源

Vercel 项目指向仓库根目录，但文件在 `openinstall-web/` 子目录中。

## ✅ 解决方案：重新链接项目

### 方法 1: 使用 CLI 重新链接（推荐，最简单）

在 `openinstall-web` 目录中运行：

```bash
cd openinstall-web

# 1. 删除旧链接
rm -rf .vercel

# 2. 重新链接（从 openinstall-web 目录链接）
npx vercel link

# 3. 选择：
#    - 选择现有项目 "openinstall-web"
#    或
#    - 创建新项目
#    
# 重要：确保在 openinstall-web 目录中运行，这样 Vercel 就知道根目录是 openinstall-web

# 4. 部署
npx vercel --prod
```

### 方法 2: 在 Dashboard 中重新导入

1. **访问**：https://vercel.com
2. **删除项目**：找到 `openinstall-web` → Settings → Delete
3. **重新导入**：
   - Add New → Project
   - 选择仓库：`twappleon/openinstall_alternative`
   - **关键**：在配置页面找到 "Root Directory" 或 "Project Root"
   - 设置为：`openinstall-web`
   - Deploy

### 方法 3: 检查项目设置的其他位置

Root Directory 可能在这些地方：
- **Settings** → **General** → 滚动到底部找 "Root Directory"
- **Settings** → **Git** → 可能有 "Root Directory" 选项
- 项目导入/编辑时的配置页面

## 🔍 为什么找不到 Root Directory？

如果 Dashboard 中没有这个选项，通常意味着：
- 项目是从仓库根目录创建的
- 需要重新创建项目才能设置
